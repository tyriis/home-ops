# Immich Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy Immich (photo/video management, no ML) in devenv for local validation, then in the main cluster.

**Architecture:** bjw-s `app-template` v5.0.1 Helm chart. Devenv has three inline controllers (server + postgres + dragonfly) with local-path PVCs and a committed plain Secret. Main cluster has a single server controller backed by a shared Dragonfly instance, a dbman-provisioned postgres17 database, and an OpenBao ExternalSecret.

**Tech Stack:** Flux CD, bjw-s app-template v5.0.1, Immich v2.x, PostgreSQL 17 (ghcr.io/immich-app/postgres), DragonflyDB, External Secrets Operator, dbman (dbman.hef.sh/v1alpha3), Envoy Gateway (HTTPRoute), kind (devenv).

**Spec:** `docs/superpowers/specs/2026-05-15-immich-design.md`
